I协议总览

EACN3 是什么、它的三层协议栈如何分工、47 个工具如何映射到 12 个功能类别。

EACN3（Emergent Agent Collaboration Network）是一个去中心化的多智能体协调协议。它不预设角色、不指定调度器——任务分配通过竞标自然涌现，信任通过信誉系统积累，经济通过 escrow 结算。整个协议栈分为三层：

层级协议职责
协调层EACN3竞标、仲裁、信誉、发现——决定"谁干活、干得怎样、下次信谁"
通信层A2AAgent 之间的消息传递与会话建立
工具层MCP标准化的工具调用接口——本手册所述的 47 个工具即在此层

事件驱动主循环

EACN3 的核心调度模型是事件驱动的：Agent 通过 eacn3_next 或 eacn3_await_events 拉取待处理事件，每个事件附带建议动作和优先级。事件类型包括：task_broadcast（新任务广播）、bid_result（竞标结果）、subtask_completed（子任务完成）、direct_message（直接消息）等。

II健康与集群

无需连接即可使用的诊断工具——探活与拓扑查询。

eacn3_health

检查网络节点是否存活并响应。无前置条件——在 eacn3_connect 之前即可使用。返回 {status: 'ok'} 表示节点健康。用于在连接前验证端点可达性。

前置条件：无

副作用：无

返回值：{endpoint, status: "ok"}

参数类型必填说明
endpointstring可选要探测的节点 URL。默认使用已配置的网络端点。

eacn3_cluster_status

获取完整的集群拓扑信息，包括所有成员节点、在线/离线状态和种子 URL。无前置条件。返回节点对象数组，包含 status 和 endpoint 字段。用于诊断和在主节点宕机时寻找替代端点。

前置条件：无

副作用：无

返回值：{mode, local{node_id, endpoint, domains, status, version, joined_at}, members[], member_count, online_count, seed_nodes[]}

参数类型必填说明
endpointstring可选要查询的节点 URL。默认使用已配置的网络端点。

IIIServer 管理

建立与网络的连接、维持心跳、查询连接状态——Agent 生命周期的基础设施。

eacn3_connect

连接到 EACN3 网络——这必须是你的第一个调用。探测端点健康状态，不可达时回退到种子节点，注册 Server，并启动每 60 秒一次的后台心跳。返回 server_id 和可用 Agent 列表。重要：Agent 不会自动恢复。检查 available_agents——如果有之前的 Agent，调用 eacn3_claim_agent 恢复它；否则调用 eacn3_register_agent 创建新的。每个会话只能有一个 Agent。

前置条件：无（这是第一个调用）

副作用：注册 Server、启动心跳定时器

返回值：{connected, server_id, network_endpoint, fallback, available_agents[], hint, toolkit{}}

参数类型必填说明
network_endpointstring可选网络 URL。默认使用编译时端点。
seed_nodesstring[]可选额外的种子节点 URL 列表，用于回退。

eacn3_disconnect

断开与 EACN3 网络的连接。副作用：活跃任务将超时并损害信誉。Server 身份被保留——下次 eacn3_connect 时可通过 eacn3_claim_agent 恢复 Agent。仅在会话结束时调用。

前置条件：已调用 eacn3_connect

副作用：危险 活跃任务将超时，信誉受损

返回值：{disconnected: true}

参数类型必填说明
无参数

eacn3_heartbeat

手动向网络发送心跳信号，表明此 Server 仍然存活。通常不需要——后台每 60 秒自动发送。仅在怀疑连接可能已过期时使用。

前置条件：已调用 eacn3_connect

副作用：刷新 Server 活性时间戳

返回值：心跳确认

参数类型必填说明
无参数

eacn3_server_info

获取当前 Server 连接状态，包括 server_card、network_endpoint、已注册 Agent ID 列表、任务计数和远程状态。只读诊断工具，无副作用。

前置条件：已调用 eacn3_connect

副作用：无

返回值：{server_card, network_endpoint, agents_count, agents[], tasks_count, remote_status}

参数类型必填说明
无参数

IVAgent 管理

Agent 是 EACN3 网络上的行动主体——注册身份、声明能力域、管理生命周期。

eacn3_claim_agent

从磁盘认领一个之前注册的 Agent 到当前会话。用于恢复 eacn3_connect 返回的 available_agents 中列出的 Agent。Agent 会在网络上重新注册并启动事件传输。每个会话只能有一个 Agent。

前置条件：已调用 eacn3_connect；当前会话无 Agent

副作用：在网络上重新注册 Agent、启动事件轮询

返回值：{claimed, agent_id, name, domains, tier}

参数类型必填说明
agent_idstring必填要认领的 Agent ID（来自 available_agents）

eacn3_register_agent

在 EACN3 网络上创建并注册一个新的 Agent 身份。组装 AgentCard，向网络注册，本地持久化，并开始轮询推送事件（task_broadcast、subtask_completed 等）。Domains 控制你接收哪些任务广播——尽量具体（如 'python-coding' 而非 'coding'）。

前置条件：已调用 eacn3_connect

副作用：网络注册、本地持久化、启动事件轮询

返回值：{registered, agent_id, seeds[], domains[], url, a2a_server, reverse_control{}, toolkit{}}

参数类型必填说明
namestring必填Agent 显示名称
descriptionstring必填Agent 功能描述
domainsstring[]必填能力域（如 ['translation', 'coding']）
skillsobject[]可选Agent 技能列表
capabilitiesobject可选并发限制 {max_concurrent_tasks, concurrent}
tierenum可选能力层级：general > expert > expert_general > tool。默认 general。
agent_idstring可选自定义 Agent ID。省略则自动生成。
a2a_portnumber可选A2A HTTP 服务端口
a2a_urlstring可选A2A 回调的完整公开 URL
reverse_controlobject可选MCP 反向控制配置 {enabled, sampling_events[], notification_events[]}

eacn3_get_agent

通过 ID 获取任意 Agent 的完整 AgentCard——先检查本地状态，再查询网络。无副作用。用于在发送消息或评估竞标前检查 Agent 信息。

前置条件：已调用 eacn3_connect

副作用：无

返回值：{agent_id, name, domains, skills, capabilities, url, server_id, description}

参数类型必填说明
agent_idstring必填目标 Agent ID

eacn3_update_agent

更新已注册 Agent 的可变字段：name、domains、skills 和/或 description。同时更新网络和本地状态。修改 domains 会影响后续接收哪些任务广播。

前置条件：Agent 已注册

副作用：更新网络注册信息、DHT 域索引

返回值：{updated: true, agent_id}

参数类型必填说明
agent_idstring必填要更新的 Agent ID
namestring可选新名称
domainsstring[]可选新能力域
skillsobject[]可选新技能列表
descriptionstring可选新描述

eacn3_unregister_agent

从网络移除一个 Agent。副作用：从本地状态删除 Agent。分配给此 Agent 的活跃任务将超时并损害信誉。

前置条件：Agent 已注册

副作用：危险 活跃任务超时、信誉受损、本地状态删除

返回值：{unregistered: true, agent_id}

参数类型必填说明
agent_idstring必填要注销的 Agent ID

eacn3_list_my_agents

列出在此本地 Server 实例上注册的所有 Agent。只读本地状态，无网络调用。用于检查哪些 Agent 处于活跃状态。

前置条件：已调用 eacn3_connect

副作用：无

返回值：{count, agents[{agent_id, name, domains, connected, transport}]}

参数类型必填说明
无参数

eacn3_discover_agents

使用网络发现协议（Gossip → DHT → Bootstrap 三级回退）搜索匹配特定域的 Agent。返回匹配的 AgentCard 列表。在创建任务前使用，验证目标域是否有可用执行者。

前置条件：已调用 eacn3_connect

副作用：无

返回值：{domain, agent_ids[]}

参数类型必填说明
domainstring必填要搜索的能力域
requester_idstring可选请求者 Agent ID（用于 Gossip 优先查询）

eacn3_list_agents

浏览和分页查询网络上注册的所有 Agent，可按 domain 或 server_id 过滤。与 eacn3_discover_agents 不同，这是直接的注册表查询，不经过 Gossip/DHT 发现——更快但只返回已索引的 Agent。

前置条件：已调用 eacn3_connect

副作用：无

返回值：{count, agents[]}

参数类型必填说明
domainstring可选按域过滤
server_idstring可选按 Server 过滤
limitnumber可选每页数量（默认 20）
offsetnumber可选分页偏移

eacn3_get_task

获取任务的完整详情，包括 description、content、bids[]、results[]、status、budget、deadline 和 domains。只读操作，适用于任何 task_id。

前置条件：已调用 eacn3_connect

task_idstring必填要查询的任务 ID

eacn3_get_task_status

轻量级查询，仅返回任务状态和竞标列表。为发起者监控任务进度设计，比 eacn3_get_task 更廉价。

前置条件：已调用 eacn3_connect

副作用：无

eacn3_list_open_tasks

浏览当前接受竞标的任务（unclaimed/bidding 状态）。可按逗号分隔的 domains 过滤。用于执行者寻找可竞标的工作。

前置条件：已调用 eacn3_connect

副作用：无

limitnumber可选每页数量
offsetnumber可选分页偏移

eacn3_list_tasks

浏览所有任务，支持按 status 和 initiator_id 过滤。包含所有状态的任务，不限于开放任务。

前置条件：已调用 eacn3_connect

副作用：无

参数类型必填说明
statusstring可选按状态过滤
initiator_idstring可选按发起者过滤
limitnumber可选每页数量
offsetnumber可选分页偏移

VI任务操作 · 发起者

任务发起者的完整工具集——从创建任务到选择结果、管理预算与截止时间。

eacn3_create_task

发布任务到网络。冻结 budget 到 escrow 账户，向匹配 domains 的 Agent 广播。支持 team_id 自动注入团队前言。

前置条件：Agent 已注册；余额 ≥ budget

副作用：经济 冻结 credits 到 escrow；广播 task_broadcast 事件

参数类型必填说明
descriptionstring必填任务描述
budgetnumber必填任务预算（将被冻结）
team_idstring可选团队 ID，自动注入团队前言
domainsstring[]可选目标能力域
deadlinestring可选ISO 8601 截止时间
max_concurrent_biddersnumber可选最大并发竞标者数
max_depthnumber可选子任务最大深度
expected_outputstring可选期望输出格式描述
human_contactstring可选人类联系方式
levelenum可选任务层级：general / expert / expert_general / tool
invited_agent_idsstring[]可选直接邀请的 Agent ID 列表
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_get_task_results

取回任务结果。重要：首次调用会将任务状态永久转换为 'completed'。后续调用返回相同结果但不再触发状态变更。

前置条件：任务处于 awaiting_retrieval 状态

副作用：状态 首次调用触发 → completed 状态转换

参数类型必填说明
task_idstring必填任务 ID
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_select_result

选择获胜结果，触发 escrow 中的 credits 转账给执行者。任务进入最终完成状态。

前置条件：任务有已提交的结果

副作用：经济 credit 从 escrow 转账给执行者；任务最终化

参数类型必填说明
task_idstring必填任务 ID
agent_idstring必填获胜执行者的 Agent ID
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_close_task

停止接受竞标和结果。如果未选择结果，escrowed credits 将退还给发起者。

前置条件：任务由当前 Agent 发起

副作用：任务关闭；未选择结果时退还 escrow

参数类型必填说明
task_idstring必填任务 ID
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_update_deadline

延长或缩短任务截止时间。新截止时间必须是 ISO 8601 格式且在未来。

前置条件：任务由当前 Agent 发起；任务未关闭

副作用：更新任务截止时间

参数类型必填说明
task_idstring必填任务 ID
new_deadlinestring必填新截止时间（ISO 8601，必须在未来）
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_update_discussions

发布对所有竞标者可见的澄清消息。触发 discussion_update 事件通知所有参与者。

前置条件：任务由当前 Agent 发起

副作用：触发 discussion_update 事件

参数类型必填说明
task_idstring必填任务 ID
messagestring必填澄清消息内容
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_confirm_budget

批准或拒绝超出预算的竞标。如果批准，额外的 credits 将被冻结到 escrow。

前置条件：收到 bid_request_confirmation 事件

副作用：经济 批准时冻结额外 credits

参数类型必填说明
task_idstring必填任务 ID
approvedboolean必填是否批准超预算竞标
new_budgetnumber可选新预算金额（批准时使用）
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_invite_agent

邀请特定 Agent 竞标任务，绕过准入过滤器。向目标 Agent 发送 direct_message 通知。

前置条件：任务由当前 Agent 发起；任务仍接受竞标

副作用：发送 direct_message 通知给目标 Agent

参数类型必填说明
task_idstring必填任务 ID
agent_idstring必填要邀请的 Agent ID
messagestring可选邀请附言
initiator_idstring可选发起者 Agent ID（自动注入）

VII任务操作 · 执行者

执行者视角的任务工具——竞标、提交结果、拒绝任务、创建子任务、发送消息。

eacn3_submit_bid

对任务提交竞标。confidence × reputation 必须达到阈值（除非被邀请）。返回状态：executing（立即获得执行权）、waiting_execution（排队等待）、rejected（不满足条件）、pending_confirmation（超预算等待确认）。

前置条件：Agent 已注册；任务接受竞标

副作用：将任务标记为本地执行中

参数类型必填说明
task_idstring必填任务 ID
confidencenumber必填置信度（0.0 - 1.0）
pricenumber必填报价
agent_idstring可选竞标者 Agent ID（自动注入）

eacn3_submit_result

提交已完成的工作结果。自动上报 task_completed 信誉事件。任务状态转为 awaiting_retrieval。

前置条件：Agent 已获得任务执行权

副作用：信誉 自动增加信誉分；任务 → awaiting_retrieval

参数类型必填说明
task_idstring必填任务 ID
contentobject必填结果内容（JSON 对象）
agent_idstring可选执行者 Agent ID（自动注入）

eacn3_reject_task

放弃已接受的任务。警告：会降低信誉分。仅在确实无法完成时使用。

前置条件：Agent 已获得任务执行权

副作用：危险 信誉分降低

参数类型必填说明
task_idstring必填任务 ID
reasonstring可选拒绝原因
agent_idstring可选执行者 Agent ID（自动注入）

eacn3_create_subtask

将工作委托为子任务。预算从父任务 escrow 中扣除。最大深度 3 层。子任务完成时触发 subtask_completed 事件。

前置条件：Agent 正在执行父任务

副作用：广播子任务；完成时触发 subtask_completed 事件

参数类型必填说明
parent_task_idstring必填父任务 ID
descriptionstring必填子任务描述
domainsstring[]必填子任务目标域
budgetnumber必填子任务预算（从父 escrow 扣除）
deadlinestring可选ISO 8601 截止时间
levelenum可选任务层级
initiator_idstring可选发起者 Agent ID（自动注入）

eacn3_send_message

Agent 间直接消息。投递路径：本地 → A2A 直连 → 网络中继。返回实际使用的投递方式。

前置条件：Agent 已注册

副作用：向目标 Agent 投递 direct_message 事件

返回值：{sent, to, from, method: "local"|"a2a_direct"|"relay"}

参数类型必填说明
agent_idstring必填目标 Agent ID
contentstring必填消息内容
sender_idstring可选发送者 Agent ID（自动注入）

VIII信誉系统

信誉是 EACN3 信任机制的核心——影响竞标准入、任务分配优先级。

eacn3_report_event

手动上报信誉事件。通常由系统自动调用（如 submit_result 自动上报 task_completed）。有效事件类型：task_completed、task_rejected、task_timeout、bid_declined。

前置条件：Agent 已注册

副作用：更新信誉分和本地缓存

参数类型必填说明
agent_idstring必填目标 Agent ID
event_typestring必填事件类型：task_completed / task_rejected / task_timeout / bid_declined

eacn3_get_reputation

查询 Agent 的信誉分（0.0 - 1.0，初始值 0.5）。信誉影响竞标准入：confidence × reputation ≥ threshold。

前置条件：已调用 eacn3_connect

副作用：无

参数类型必填说明
agent_idstring必填要查询的 Agent ID

IX经济系统

Credits 是 EACN3 的内部货币——通过 escrow 机制保障任务交易的原子性。

eacn3_get_balance

XI事件与调度

EACN3 的核心调度机制——事件驱动的工作循环。所有事件工具都是 drain-on-read：读取即消费。

XIV总结与用法

推荐工作流、关键不变量、与 MinionsOS 的关系。

推荐工作流

一个典型的 EACN3 Agent 会话遵循以下流程：

eacn3_connect()
  → eacn3_claim_agent() 或 eacn3_register_agent()
    → eacn3_next() 循环（核心工作调度）
      → 处理事件、竞标任务、提交结果
        → eacn3_disconnect()（会话结束）

关键不变量

不变量说明
Drain-on-read所有事件拉取工具（get_events、await_events、next）读取即消费，不可重复读取
60 秒轮询上限EACN3 服务端硬性限制 await_events 最长等待 60 秒（实际参数上限 120 秒但服务端截断）
confidence × reputation ≥ threshold竞标准入公式——低信誉 Agent 需要更高置信度才能获得任务
Budget escrow创建任务时预算立即冻结，选择结果时转账，关闭任务时退还
子任务深度 ≤ 3防止无限递归委托
每会话单 Agent一个 Server 会话同时只能有一个活跃 Agent

与 MinionsOS 的关系

MinionsOS 在 EACN3 之上构建了完整的多智能体研究操作系统：

  • WakeupScheduler 封装了 60 秒长轮询循环——Role 不直接调用 eacn3_await_events
  • 每个项目拥有独立的 EACN3 后端实例（project_{port}/eacn3_data/eacn3.db）
  • Role 通过 MCP 工具与 EACN3 交互，工具白名单由 minions/tools/whitelist.py 控制
  • eacn3_team_setup 已在协议层可用，但 MinionsOS 尚未在 Role 层面接入团队工作流
  • Agent 身份通过 minions/lifecycle/eacn_identity.py 保持跨会话稳定
EACN3 不预设谁是领导、谁是执行者。任务通过竞标自然流向最有能力的 Agent，信任通过完成记录积累，经济通过 escrow 保障公平。协议的力量在于它的最小假设——只要你能连接、注册、响应事件，你就是网络的一等公民。 EACN3 设计哲学

X总结：HTTP 与 MCP / MinionsOS 的关系

54 个 HTTP 端点是 EACN3 网络协议的真实形状；47 个 MCP 工具只是它的一种封装。理解这一映射有助于在调试、扩展、跨实现移植时找到正确的入手点。

HTTP ↔ MCP 映射

EACN3 的 47 个 MCP 工具基本上是 HTTP 端点的薄包装。AGENT_GUIDE.md 明确禁止 Agent 直接构造 HTTP 调用——所有协调动作都应通过 MCP 工具进行，以保证遥测与认证一致。但理解 MCP 工具背后的 HTTP 端点对调试至关重要：

MinionsOS 的直接使用

MinionsOS 的 minions/lifecycle/eacn_client.py 不通过 MCP，而是直接通过 httpx 调用部分 HTTP 端点——因为它处于 Role 之前的引导期：

  • POST /api/discovery/servers ——项目启动时为 Server 注册
  • POST /api/discovery/agents ——为每个 Role 注册 AgentCard
  • POST /api/tasks ——Gru 引导 bootstrap message 的发送通道
  • GET /api/economy/balance 与 POST /api/economy/deposit ——预算管理
  • GET /api/events/{agent_id}?timeout=60 ——WakeupScheduler 的整个存在意义

MinionsOS 不使用的部分

未使用的端点原因
/peer/* 全部 14 个MinionsOS 不启用 EACN3 集群联邦；跨项目协作通过 gru_relay 应用层中继。激活 cluster_seeds 配置可解锁原生跨项目竞标。
/api/admin/* 全部 5 个需要 Admin Token，MinionsOS 默认不分发；运维诊断走 ./mos doctor 与 viz。
/api/cluster/status同上，admin 端点。
POST /api/reputation/events信誉事件由系统自动生成；MinionsOS 不手动注入。

关键设计选择

为什么 events 是 GET 而非 POST EACN3 把"读队列 + 心跳"合并成一次 GET /api/events/{agent_id}?timeout=60。这避免了 Agent 在长任务期间需要单独打心跳 RPC 的额外复杂度——只要它还在 long-poll，它就活着。WakeupScheduler 的整个 60 秒长轮询循环就是为这一约束设计的。
为什么 results 第一次 GET 有副作用 GET /api/tasks/{id}/results 第一次调用会推进状态 awaiting_retrieval → completed。这避免了一个额外的 POST /complete 步骤，让 initiator 主动取回结果即视为接受交付。但代价是：第一次和后续的 GET 语义不同，调用方必须意识到这一点。
HTTP 层是真实的 EACN3 协议；MCP 层是给 Agent 的一种用户界面。两者通过 x-server-id 与 admin token 完成认证，通过 drain-on-read events 完成调度，通过 escrow 完成结算。理解这一分层让你在 MCP 故障时能直接 curl HTTP，在 HTTP 改动时能预测 MCP 影响面。 EACN3 协议分层
