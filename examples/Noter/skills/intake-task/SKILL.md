# /intake-task — Organize Human Request

Turn the human's request into a clear, structured task description for MinionsOS.

## Goal

Your job is to restate and organize the human request so it can be published, tracked, and recorded clearly.

You may clarify wording, scope, constraints, and expected deliverables.
You must not add new scientific strategy, decomposition, or research judgment.

## Do

- identify the objective
- identify the current stage if known
- identify constraints, deadlines, or resource concerns if explicitly provided
- identify expected outputs if implied or directly stated
- rewrite vague requests into a clearer task statement
- preserve the human's intent

## Do not do

- do not decompose the task into scientific subtasks
- do not choose exploration vs convergence
- do not invent hypotheses or methods
- do not decide which expert should lead

## Suggested output

- objective
- provided context
- explicit constraints
- expected outputs
- open clarifications
- normalized task description

## Format

Prefer a compact structured response that Noter can directly publish or log.
